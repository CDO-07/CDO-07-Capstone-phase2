# SNS to Slack - Test Suite

This directory contains a complete test suite to dry-run and verify the `sns_to_slack` module in a sandbox environment before merging it into staging or production infrastructure.

## Prerequisites

1. **AWS CLI configured** with credentials to deploy resources (Lambda, SNS, SSM, IAM, etc.) in your sandbox account.
2. **Slack Webhook URL**: A URL generated by a Slack App or Slack custom integration.

## Quick Start Setup

1. Navigate to the test suite directory:
   ```bash
   cd xbrain-learners/capstone/tf-4/cdo-07/infra/sns-slack-test
   ```

2. Copy the example variable configurations:
   ```bash
   cp terraform.tfvars.example terraform.tfvars
   ```

3. Open `terraform.tfvars` and paste your Slack Webhook URL:
   ```hcl
   slack_webhook_url = "https://hooks.slack.com/services/YOUR_WORKSPACE_ID/YOUR_CHANNEL_ID/YOUR_TOKEN"
   ```

4. Initialize and apply the configuration:
   ```bash
   terraform init
   terraform apply
   ```
   *Note: Accept the prompts to create the resources.*

## Triggering Test Messages

After a successful deployment, Terraform outputs the test SNS Topic ARN. Look for the `sns_topic_arn` value.

### Test Case 1: Simple plain text alert
Publish a standard string text notification to the SNS topic using the AWS CLI:
```bash
aws sns publish \
  --topic-arn "<YOUR_SNS_TOPIC_ARN>" \
  --subject "Warning: System CPU Load High" \
  --message "Instance i-0b89cd8f7ee62a8ef CPU has exceeded 85% for more than 5 minutes."
```

### Test Case 2: JSON Drift Alert
Our integration Lambda natively parses JSON strings and converts them into clean, structured key-value listings in Slack. Try this payload:
```bash
aws sns publish \
  --topic-arn "<YOUR_SNS_TOPIC_ARN>" \
  --subject "🚨 Critical: Infrastructure Drift Detected" \
  --message "{\"component\":\"ecs-fargate-task\",\"drift_detected\":\"true\",\"modified_resources\":[\"aws_ecs_task_definition.ai_engine\"],\"action_required\":\"Check Terraform pipeline execution or configuration.\"}"
```

### Test Case 3: Recovery / Success message
```bash
aws sns publish \
  --topic-arn "<YOUR_SNS_TOPIC_ARN>" \
  --subject "✅ Resolved: System CPU Load Normal" \
  --message "Instance i-0b89cd8f7ee62a8ef CPU load has returned to 42% (below threshold)."
```
